## 🎯 Простыми словами: кто есть кто

Представьте **сетевой ресторан** с доставкой:

| Уровень | Аналогия | Что делает |
|---------|----------|------------|
| **CLOUD LAYER** | Головной офис (сайт, колл-центр) | Хранит всех клиентов, менеджеров, цены. Работает всегда и везде |
| **CLUB LAYER** | Локальный менеджер в конкретном ресторане | Следит за залами, столиками, электричеством. Работает даже если интернет пропал |
| **PC CLIENT** | Официант + сам столик | Показывает меню, принимает заказ, следит за временем |

## 📋 Бизнес-процесс: как клиент пришел играть

### ШАГ 1: Клиент заходит в клуб

**Что происходит:**

1. **Клиент** подходит к **кассиру** (у кассира открыт веб-интерфейс CLOUD LAYER)
2. Кассир вводит номер телефона клиента
3. CLOUD LAYER говорит: "У клиента 500 рублей на счету"

```javascript
// Кассир нажимает "Пополнить баланс"
CLOUD LAYER (Laravel) → 
  Проверяет клиента в БД → 
  Отвечает: "Баланс 500 руб"
```

**Почему нельзя без CLOUD LAYER?**  
Потому что клиент мог пополнить счет дома через сайт, и эти данные хранятся в облаке.

---

### ШАГ 2: Клиент выбирает компьютер

**Что происходит:**

1. Кассир выбирает **свободный ПК** (например, №5)
2. CLOUD LAYER отправляет команду на **ClubServer** (который стоит в локальной сети клуба)
3. ClubServer будит компьютер и говорит: "Клиент Иванов будет сейчас играть"

```javascript
// 1. Кассир в браузере нажимает "Начать сеанс на ПК №5"
CLOUD LAYER → WebSocket → ClubServer

// 2. ClubServer получает команду
ClubServer → WebSocket → ПК №5
```

**Зачем здесь ClubServer?**  
Потому что CLOUD LAYER **не может напрямую** обратиться к компьютеру №5:
- У компьютера нет внешнего IP-адреса (он за роутером)
- Компьютер может быть выключен (ClubServer разбудит через Wake-on-LAN)

---

### ШАГ 3: На компьютере открывается форма авторизации

**Что происходит:**

1. На ПК №5 запущено **Electron-приложение** (LoginApp)
2. Приложение получает команду от ClubServer
3. На экране появляется: "Введите код" или автоматический вход

```javascript
// Electron приложение на ПК №5 постоянно слушает ClubServer
PC Client (WebSocket) ← ClubServer: "Авторизуй клиента Иванова"
PC Client → показывает экран входа
```

---

### ШАГ 4: Клиент играет, идет списание денег

**Самый важный момент!** Списание денег происходит **локально**, чтобы не нагружать интернет:

```javascript
// НА ПК №5 (Electron приложение)
Каждую секунду:
  баланс = баланс - 0.10 руб (цена тарифа)
  
  if (баланс <= 0) {
    // Деньги кончились - блокируем компьютер
    блокируем_экран();
    отправляем_ClubServer("Сессия завершена");
  }
  
  if (баланс <= 30 руб) {
    // За 5 минут до окончания
    показываем_предупреждение("Осталось 5 минут!");
  }
```

**Что делает ClubServer в это время:**  
Каждые 5 минут сохраняет состояние в SQLite:  
`"ПК №5, сессия активна, остаток 450 руб"`

**Почему списание на ПК, а не в облаке?**  
Представьте, что у 50 компьютеров идет списание каждую секунду. Если отправлять каждую секунду запрос в облако:
- Нужен **очень мощный сервер**
- Если **интернет пропадет** - все компьютеры перестанут списывать деньги (катастрофа!)

Поэтому списание **локальное**, а с облаком синхронизируемся раз в 5-10 минут.

---

### ШАГ 5: Клиент решил купить колу в баре

**Что происходит:**

1. Кассир нажимает "Пополнить баланс" в своем веб-интерфейсе
2. CLOUD LAYER обновляет баланс в БД
3. CLOUD LAYER отправляет **WebSocket** сообщение в ClubServer
4. ClubServer передает команду на ПК №5
5. На экране клиента появляется: "Баланс пополнен на 200 руб"

```javascript
// Кассир в браузере
CLOUD LAYER: UPDATE balance SET amount = 500 WHERE user_id = 123
CLOUD LAYER → WebSocket → ClubServer: "У клиента 123 новый баланс 500"

// ClubServer получает
ClubServer → WebSocket → ПК №5: "Обнови баланс до 500"

// ПК №5
Electron приложение обновляет счетчик на экране
```

## 🔌 Что если пропал интернет?

Это самое крутое в такой архитектуре!

### Ситуация: Интернет в клубе отключился

```javascript
// ClubServer замечает: связь с CLOUD LAYER потеряна
ClubServer: "Ой, интернета нет. Включаю офлайн-режим"

// Что происходит:
1. Новые клиенты НЕ могут авторизоваться (не проверить баланс)
2. ТЕКУЩИЕ клиенты продолжают играть (списание локальное)
3. Все транзакции сохраняются в SQLite на ClubServer
4. Через 120 минут - принудительное завершение всех сессий
```

**Код в ClubServer для офлайн-режима:**
```javascript
class OfflineMode {
  async handle() {
    // Кэшируем транзакции
    this.cache.save({
      computerId: 5,
      action: 'start_session',
      clientId: 123,
      timestamp: Date.now()
    });
    
    // Проверяем таймаут
    if (this.offlineDuration > 120 * 60 * 1000) {
      this.forceEndAllSessions(); // Через 2 часа - всем выход
    }
  }
  
  async syncWhenOnline() {
    // Интернет появился - отправляем все накопленные транзакции
    const cached = await this.cache.getAll();
    await axios.post('https://cloud/api/sync', cached);
  }
}
```

## 🎮 А как с Play Station?

Play Station **нельзя установить** наше приложение (это не Windows).

Поэтому схема другая:

```javascript
// Кассир нажимает "Включить PS5"
CLOUD LAYER → ClubServer: "Включи PS5"

// ClubServer знает IP-адрес Web Power Switch (устройство, которое управляет розетками)
ClubServer → WPS (http://192.168.1.100): "Включи розетку №3"

// WPS подает питание на PS5
PS5 включается
```

**Почему так сложно?**  
PS5 нельзя управлять программно (нет API), поэтому управляем **розеткой** через WPS.

## 💡 Итог: зачем 3 уровня?

| Если бы было 2 уровня | Почему плохо |
|----------------------|--------------|
| **Cloud + PC напрямую** | 1. Нет локального офлайн-режима<br>2. Каждое списание = запрос в интернет<br>3. Нельзя управлять выключенными ПК<br>4. 50 ПК * 1 запрос/сек = сервер умрет |

| С 3 уровнями | Преимущества |
|-------------|--------------|
| **Cloud + ClubServer + PC** | 1. Офлайн-режим (интернет пропал - играем дальше)<br>2. Списание локальное (не грузит сервер)<br>3. ClubServer будит ПК через локальную сеть<br>4. ClubServer управляет розетками для PS5 |

## 🔧 Простой пример кода

**Как ClubServer общается с Cloud:**
```javascript
// ClubServer -> Cloud (раз в 5 минут)
setInterval(() => {
  const sessions = getActiveSessions(); // [{pc:5, balance:450}]
  
  fetch('https://cloud-api.com/sync', {
    method: 'POST',
    body: JSON.stringify(sessions)
  }).catch(() => {
    // Интернет пропал - сохраняем в SQLite
    saveToSQLite(sessions);
  });
}, 300000);
```

**Как ClubServer общается с ПК:**
```javascript
// ПК подключается к ClubServer при загрузке Windows
const ws = new WebSocket('ws://192.168.1.200:3000');

// ClubServer говорит ПК заблокироваться
ws.on('message', (msg) => {
  if (msg.type === 'lock') {
    blockScreen(); // Electron киоск-режим
  }
});
```

**Как ПК списывает деньги:**
```javascript
// На ПК №5
let balance = 500;
setInterval(() => {
  balance -= 0.10; // списываем копейку в секунду
  updateScreen(balance);
  
  if (balance <= 0) {
    lockComputer();
    ws.send('session_ended');
  }
}, 1000);
```

Теперь понятно? Это классическая **трехуровневая архитектура** для систем, где важна:
1. **Надежность** (офлайн-режим)
2. **Производительность** (локальные вычисления)
3. **Безопасность** (локальное управление ПК)

Cloud Layer = голова (стратегия)  
Club Layer = спинной мозг (локальная координация)  
PC Client = мышцы (исполнение)

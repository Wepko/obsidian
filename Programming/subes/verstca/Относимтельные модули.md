## Отличная мысль! Вы абсолютно правы — грамотное использование относительных единиц сильно сокращает количество медиазапросов

Давайте разберем, как и где применять каждую единицу, чтобы сделать верстку максимально адаптивной.

---

## 1. Краткий справочник единиц

| Единица | Относительно чего | Когда использовать |
|---------|-------------------|-------------------|
| **rem** | Корневой шрифт (`<html>`), по умолчанию 16px | Размеры текста, отступы, ширина контейнеров |
| **em** | Размер шрифта родительского элемента | Паддинги/марджины внутри компонента |
| **%** | Родительский элемент | Ширина блоков, flex/grid элементы |
| **vw** | 1% от ширины окна | Крупные отступы, размеры шрифтов для заголовков |
| **vh** | 1% от высоты окна | Высота секций, модальные окна |
| **vmin** | Меньшая из vw и vh | Шрифты для мобильных (квадратные экраны) |
| **vmax** | Большая из vw и vh | Редко, для специфичных эффектов |
| **ch** | Ширина символа "0" | Ограничение длины строки текста |
| **ex** | Высота символа "x" | Редко, для вертикальных ритмов |

## 2. Стратегия: минимизация медиазапросов

### Базовые настройки (один раз на весь проект)

```scss
// _variables.scss
// База — 16px, но можно менять под свои нужды
html {
  font-size: 16px;
  
  // На планшетах база чуть меньше
  @media (max-width: 768px) {
    font-size: 14px;
  }
  
  // На мобильных еще меньше
  @media (max-width: 480px) {
    font-size: 12px;
  }
}

// Все размеры теперь в rem
body {
  font-size: 1rem;      // 16px на десктопе, 14px на планшете, 12px на мобильном
  line-height: 1.5;
}

.container {
  max-width: 75rem;     // 1200px на десктопе
  margin: 0 auto;
  padding: 0 1rem;
}
```

**Результат:** Изменяя `font-size` на `html`, вы масштабируете весь сайт.

---

### 3. Отступы и размеры — на rem

```scss
// Все отступы в rem
.card {
  padding: 1.5rem;      // 24px на десктопе, 21px на планшете, 18px на мобильном
  margin-bottom: 2rem;  // 32px → 28px → 24px
  border-radius: 0.5rem;
}

.section {
  padding: 4rem 0;      // 64px → 56px → 48px
}

// Заголовки — тоже в rem
h1 {
  font-size: 3rem;      // 48px → 42px → 36px
  margin-bottom: 1rem;
}

h2 {
  font-size: 2rem;      // 32px → 28px → 24px
}
```

**Плюс:** Не нужно писать медиазапросы для каждого элемента — всё масштабируется автоматически.

---

### 4. Гибкие сетки через % и fr

```scss
// Сетка проектов — адаптируется без медиазапросов
.projects-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 1.5rem;
}

// Или с фиксированным количеством колонок, но с % шириной
.features {
  display: flex;
  flex-wrap: wrap;
  gap: 1.5rem;
  
  &__item {
    flex: 1;
    min-width: 250px;
    // Ширина автоматически подстраивается
  }
}
```

**Результат:** Колонки сами адаптируются под ширину экрана.

---

### 5. Шрифты через clamp() — мощный инструмент

```scss
// clump(min, ideal, max)
h1 {
  font-size: clamp(1.75rem, 5vw, 3rem);
  // на телефоне: 28px
  // на планшете: плавно растет
  // на десктопе: 48px
}

h2 {
  font-size: clamp(1.5rem, 4vw, 2.25rem);
}

body {
  font-size: clamp(0.875rem, 2vw, 1rem);
}

// Отступы тоже можно через clamp
.section {
  padding: clamp(2rem, 8vw, 6rem) 0;
}
```

**Результат:** Адаптация происходит плавно, без резких скачков на медиазапросах.

---

### 6. Типографика через vw (но с ограничениями)

```scss
// Вариант: шрифт масштабируется с экраном
h1 {
  font-size: calc(24px + 2vw);
  // на 375px: 24 + 7.5 = 31.5px
  // на 1200px: 24 + 24 = 48px
}

// Лучше с ограничениями через min/max
h1 {
  font-size: max(1.75rem, 4vw);
  // минимум 28px, но растет с экраном
}

// Или через clamp (современный подход)
h1 {
  font-size: clamp(1.75rem, 5vw, 3rem);
}
```

---

### 7. Высота секций через vh

```scss
.hero {
  min-height: 70vh;     // 70% высоты экрана
  display: flex;
  align-items: center;
  
  @media (max-width: 768px) {
    min-height: 50vh;   // на мобильных меньше
  }
}

.fullscreen {
  height: 100vh;        // на всю высоту экрана
}

.modal {
  max-height: 80vh;
  overflow-y: auto;
}
```

---

### 8. Ширина контейнеров

```scss
.container {
  width: 100%;
  max-width: 75rem;     // 1200px
  margin: 0 auto;
  padding: 0 1rem;
  
  // Для очень больших экранов
  @media (min-width: 1600px) {
    max-width: 90rem;   // 1440px
  }
}

// Альтернатива: ширина в % с ограничением
.content {
  width: min(90%, 1200px);
  margin: 0 auto;
}
```

## 9. Полный пример: карточка проекта без медиазапросов

```scss
.project-card {
  // Все размеры в rem
  padding: 1rem;
  border-radius: 0.5rem;
  
  &__image {
    width: 100%;
    aspect-ratio: 4 / 3;        // Соотношение сторон вместо фиксированной высоты
    object-fit: cover;
  }
  
  &__title {
    font-size: clamp(1.125rem, 2vw, 1.5rem);
    margin: 0.75rem 0;
  }
  
  &__info {
    display: flex;
    gap: 1rem;
    font-size: 0.875rem;
    
    span {
      padding: 0.25rem 0.5rem;
      background: #f0f0f0;
      border-radius: 0.25rem;
    }
  }
  
  &__link {
    display: inline-block;
    margin-top: 1rem;
    font-size: 0.875rem;
    padding: 0.5rem 1rem;
  }
}

// Сетка карточек — адаптивная без медиазапросов
.projects-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: clamp(1rem, 2vw, 2rem);
}
```

**Этот код работает на любых экранах без единого медиазапроса!**

## 10. Когда медиазапросы всё равно нужны

Даже с относительными единицами, медиазапросы остаются для:

| Сценарий | Пример |
|----------|--------|
| **Смена структуры** | Меню становится вертикальным |
| **Скрытие блоков** | Скрыть декоративные элементы на мобильных |
| **Смена поведения** | Слайдер превращается в сетку |
| **Точные настройки** | Максимальная ширина контейнера на больших экранах |

```scss
// Медиазапросы только для изменения структуры, не для размеров
.menu {
  display: flex;
  
  @media (max-width: 768px) {
    flex-direction: column;      // структура, не размер
    position: fixed;
    top: 0;
    right: -100%;
    width: 80%;
    height: 100vh;
    transition: right 0.3s;
    
    &.active {
      right: 0;
    }
  }
}
```

## 11. Итоговая стратегия

### Что использовать для чего:

| Элемент | Единица | Почему |
|---------|---------|--------|
| **Шрифты текста** | `rem` + `clamp()` | Масштабируются с базовым размером |
| **Заголовки** | `clamp()` с vw | Плавно растут с экраном |
| **Отступы внутри компонента** | `rem` | Масштабируются с контентом |
| **Отступы между компонентами** | `rem` или `clamp()` | Единая система |
| **Ширина блоков** | `%` или `fr` | Гибкая сетка |
| **Высота секций** | `vh` | Зависит от экрана |
| **Изображения** | `%` + `aspect-ratio` | Сохраняют пропорции |
| **Контейнер** | `min(90%, 1200px)` | Ширина с ограничением |
| **Сетки** | `auto-fill` + `minmax()` | Автоматическая адаптация |

## Итог

| Подход | Количество медиазапросов | Сложность поддержки |
|--------|-------------------------|---------------------|
| **Пиксели (px)** | Много (на каждый брейкпоинт) | Высокая |
| **rem + медиазапросы** | Среднее (на смену структуры) | Средняя |
| **clamp() + % + auto-fill** | Минимум (только на структуру) | Низкая |

**Ваша идея абсолютно верна:** современная верстка на относительных единицах + `clamp()` + гибкие сетки сокращает количество медиазапросов на 70–80%. Медиазапросы остаются только для изменения **структуры** (меню, скрытие блоков), а не для размеров.

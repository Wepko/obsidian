# 1  Тестовое задание: PathVoice Core — интеграция с AI-сервисами

## Контекст

Вы разрабатываете центральный оркестратор для PathVoice — **PHP-сервис**, который управляет коммуникациями между клиентами, AI-агентами и внешними системами.

**Реальная архитектура PathVoice:**
- **PHP-сервис (ваш)** — принимает запросы, управляет лидами, сценариями, оркестрирует запуск AI-агентов.
- **AI-сервисы (Python)** — занимаются обработкой голоса, диалогами, синтезом речи.
- **Внешние сервисы** — CRM, телефония, колл-центры.

## Задание

Разработать прототип оркестратора, демонстрирующий взаимодействие между PHP и AI-сервисами.

### Функциональные требования

#### 1. **AI-сервисы (имитация)**
Создать два упрощённых мок-сервиса на PHP (или встроить их в основной проект как отдельные классы), которые имитируют поведение реальных Python-сервисов компании:

**А. Conversation Bot (AI-диалог)**
- Эндпоинт: `POST /api/ai/conversation`
- Принимает: `{ "lead_id": 123, "message": "текст от клиента", "context": {...} }`
- Возвращает: 
  ```json
  {
    "response_text": "Спасибо за обращение, мы напомним о платеже",
    "sentiment": "positive",
    "action": "remind_about_payment",
    "confidence": 0.95
  }
  ```

**Б. Virtual Manager (принятие решений)**
- Эндпоинт: `POST /api/ai/decide`
- Принимает: `{ "lead_id": 123, "conversation_history": [...], "business_rules": {...} }`
- Возвращает:
  ```json
  {
    "next_action": "schedule_call|transfer_to_operator|send_sms|close_case",
    "priority": 1,
    "reasoning": "Клиент готов общаться, но хочет перенести разговор"
  }
  ```

*Важно: В реальном проекте это отдельные Python-микросервисы. Сейчас можно реализовать их как контроллеры внутри Laravel, чтобы показать понимание контрактов взаимодействия.*

#### 2. **Основной оркестратор (то, что нужно разработать)**

**Сценарий: Обработка входящего звонка**

Создать эндпоинт: `POST /api/incoming-call`

**Что должен делать оркестратор:**

1. **Принять запрос от телефонии:**
   ```json
   {
     "call_id": "CALL_123",
     "phone": "+1234567890",
     "transcript": "Алло, я по поводу долга, когда мне платить?",
     "call_time": "2026-03-10T15:30:00Z"
   }
   ```

2. **Найти или создать лида** по номеру телефона (использовать firstOrCreate).

3. **Запросить контекст** (история взаимодействий, сумма долга, статус).

4. **Вызвать AI-сервисы последовательно:**
   - Сначала `Conversation Bot` — получить понимание запроса.
   - Потом `Virtual Manager` — принять решение о следующем действии.

5. **Выполнить решение**:
   - Если `transfer_to_operator` — создать задачу для колл-центра.
   - Если `schedule_call` — запланировать обратный звонок.
   - Если `send_sms` — отправить уведомление (имитация).

6. **Вернуть ответ** телефонии:
   ```json
   {
     "call_id": "CALL_123",
     "action": "transfer_to_operator",
     "response_text": "Соединяю с оператором...",
     "priority_queue": "high"
   }
   ```

#### 3. **Асинхронная обработка**

- Все вызовы AI-сервисов должны быть **асинхронными через очереди** (RabbitMQ/Redis).
- Использовать **отложенные джобы** для повторных звонков.
- Реализовать **retry-механизм** с exponential backoff при падении AI-сервисов.

#### 4. **gRPC-взаимодействие (опционально, но приветствуется)**

Реализовать один из AI-сервисов как **gRPC-сервер** (можно на PHP с помощью `spiral/roadrunner-grpc` или на Python для большей реалистичности).

*Почему это важно:* Мы переходим на gRPC для межсервисного взаимодействия вместо REST.

#### 5. **Сбор метрик для Datadog**

В ключевых точках (вызов AI, решение менеджера, перевод на оператора) отправлять "метрики":
- Через **лог в формате Datadog**:
  ```
  [METRIC] call_ai_duration:245ms service:conversation_bot result:success
  ```
- Или через **фейковый HTTP-клиент** к Datadog API (можно просто логировать).

#### 6. **Поиск логов в OpenSearch**

Создать команду Artisan `php artisan logs:search --call-id=CALL_123`, которая имитирует поиск в OpenSearch (читает из файла или БД, но структура данных должна отражать понимание, как хранятся логи в распределённой системе).

## Документация и примеры

### Пример: Простой бот для техподдержки

```yaml
# scenario.yaml
name: "Tech Support Bot"
triggers:
  - incoming_call
  - incoming_sms

steps:
  - id: greeting
    type: ai_response
    prompt: "Приветствуйте клиента и спросите, как помочь"
    next: understand_intent

  - id: understand_intent
    type: ai_decision
    options:
      - intent: "technical_issue"
        next: collect_details
      - intent: "billing_question"
        next: transfer_to_billing
      - intent: "talk_to_human"
        next: transfer_to_operator

  - id: collect_details
    type: ask_question
    question: "Опишите проблему подробнее"
    store_as: issue_description
    next: resolve_or_escalate
```

Или программно:

```php
$scenario = Scenario::create('tech_support')
    ->onIncomingCall(function ($call) {
        $intent = $call->askAI("Определите интент: " . $call->transcript);
        
        return match($intent) {
            'technical' => $call->transferToQueue('tech_support'),
            'billing' => $call->transferToQueue('billing'),
            default => $call->say("Соединяю с оператором")->transferToAgent()
        };
    });
```

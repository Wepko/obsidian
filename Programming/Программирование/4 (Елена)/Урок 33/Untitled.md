# Конспект: Коллекции (Контейнеры) данных в Python

В Python существует несколько встроенных типов данных, которые используются для хранения коллекций (наборов) элементов. Главные из них: **Списки (list)**, **Кортежи (tuple)** и **Словари (dict)**. Также часто рассматривают **Множества (set)**.

Основное различие между ними заключается в том, **изменяемы** ли они, **упорядочены** ли и допускают ли **повторения** элементов.

---

## 1. Список (List)

Список — это упорядоченная, изменяемая коллекция, которая допускает дубликаты. Это самый гибкий и часто используемый тип коллекции в Python.

*   **Изменяемость (Mutable):** **Да**. Можно изменять, добавлять и удалять элементы после создания.
*   **Упорядоченность:** **Да**. Элементы хранятся в том порядке, в котором мы их добавили. Доступ к ним возможен по индексу (начиная с 0).
*   **Дубликаты:** **Да**.

**Синтаксис:** Создается с помощью квадратных скобок `[]`.

**Пример:**
```python
my_list = [1, 2, 3, "Привет", 3.14, 2] # Смесь типов и дубликат "2"
print(my_list[0]) # Вывод: 1 (доступ по индексу)
my_list[1] = 100  # Изменяем элемент
print(my_list)    # Вывод: [1, 100, 3, 'Привет', 3.14, 2]
my_list.append("Новый элемент") # Добавляем в конец
print(my_list)
# Вывод: [1, 100, 3, 'Привет', 3.14, 2, 'Новый элемент']
```

**Когда использовать:** Когда вам нужна упорядоченная последовательность элементов, которую вы планируете изменять (добавлять, удалять, сортировать).

---

## 2. Кортеж (Tuple)

Кортеж — это упорядоченная, **неизменяемая** коллекция, которая допускает дубликаты. Его часто называют "защищенным списком".

*   **Изменяемость (Mutable):** **Нет (Immutable)**. Нельзя изменить, добавить или удалить элементы после создания кортежа. Это делает код безопаснее и иногда быстрее.
*   **Упорядоченность:** **Да**. Как и список, хранит порядок элементов. Доступ по индексу.
*   **Дубликаты:** **Да**.

**Синтаксис:** Создается с помощью круглых скобок `()`.

**Пример:**
```python
my_tuple = (1, 2, 3, "Привет", 2)
print(my_tuple[0]) # Вывод: 1
print(my_tuple.count(2)) # Вывод: 2 (методы есть, но не изменяющие)
# my_tuple[1] = 100 # ОШИБКА! Кортеж не поддерживает изменение элементов

# Можно создать кортеж без скобок (упаковка кортежа)
another_tuple = 10, 20, 30
print(another_tuple) # Вывод: (10, 20, 30)
```

**Когда использовать:**
*   Для данных, которые не должны меняться (например, координаты точки `(x, y)`, дни недели, настройки конфигурации).
*   В качестве ключей в словарях (списки ключами быть не могут, так как они изменяемы).

---

## 3. Словарь (Dictionary)

Словарь — это неупорядоченная (до Python 3.6) / упорядоченная (с Python 3.7+), изменяемая коллекция, которая хранит пары **"ключ: значение"**.

*   **Изменяемость (Mutable):** **Да**. Можно менять значения, добавлять и удалять пары ключ-значение.
*   **Упорядоченность:** **Да (с Python 3.7)**. Запоминают порядок вставки элементов.
*   **Дубликаты:** **Ключи — Нет, Значения — Да**. Ключи должны быть уникальными (и неизменяемыми), значения могут повторяться.
*   **Доступ:** Доступ к элементам происходит по **ключу**, а не по индексу.

**Синтаксис:** Создается с помощью фигурных скобок `{}`, где ключ и значение разделяются двоеточием `:`.

**Пример:**
```python
my_dict = {
    "name": "Анна",   # Ключ "name", значение "Анна"
    "age": 25,        # Ключ "age", значение 25
    "city": "Москва"  # Ключ "city", значение "Москва"
}

print(my_dict["name"]) # Вывод: Анна (доступ по ключу)
my_dict["age"] = 26    # Изменяем значение
my_dict["job"] = "Инженер" # Добавляем новую пару
print(my_dict)
# Вывод: {'name': 'Анна', 'age': 26, 'city': 'Москва', 'job': 'Инженер'}
```

**Когда использовать:** Когда нужно быстро найти значение по уникальному идентификатору (ключу). Например: информация о пользователе по его ID, телефонная книга (имя -> номер), конфигурации.

---

## 4. Множество (Set) (для дополнения картины)

Множество — это неупорядоченная, изменяемая коллекция уникальных элементов.

*   **Изменяемость (Mutable):** **Да**. Можно добавлять и удалять элементы.
*   **Упорядоченность:** **Нет**. Элементы хранятся в произвольном порядке, нельзя обратиться по индексу.
*   **Дубликаты:** **Нет**. Автоматически хранит только уникальные значения.

**Синтаксис:** Создается с помощью фигурных скобок `{}`, но без двоеточий (в отличие от словаря).

**Пример:**
```python
my_set = {1, 2, 3, 4, 2, 1} # Дубликаты 2 и 1 будут проигнорированы
print(my_set) # Вывод: {1, 2, 3, 4} (порядок может отличаться)

my_set.add(5)
my_set.add(2) # 2 уже есть, ничего не изменится
print(my_set) # Вывод: {1, 2, 3, 4, 5}

# Используется для математических операций
set_a = {1, 2, 3}
set_b = {3, 4, 5}
print(set_a.intersection(set_b)) # Вывод: {3} (пересечение)
```

**Когда использовать:** Для хранения набора уникальных значений (теги, посетители сайта) и для выполнения математических операций над множествами (объединение, пересечение, разность).

## Сравнительная таблица (Шпаргалка)

| Характеристика     | Список (list)               | Кортеж (tuple)               | Словарь (dict)               | Множество (set)              |
| :----------------- | :--------------------------- | :---------------------------- | :---------------------------- | :---------------------------- |
| **Синтаксис**      | `[1, 2, 3]`                  | `(1, 2, 3)` или `1, 2, 3`     | `{"a":1, "b":2}`              | `{1, 2, 3}`                   |
| **Изменяемость**   | **Да** (Mutable)             | **Нет** (Immutable)           | **Да** (Mutable)              | **Да** (Mutable)              |
| **Упорядоченность**| **Да** (по индексу)          | **Да** (по индексу)           | **Да** (по порядку вставки)*  | **Нет** (неупорядочено)       |
| **Дубликаты**      | **Да**                       | **Да**                        | **Ключи: нет / Знач: да**     | **Нет** (только уникальные)   |
| **Доступ**         | По индексу (`list[0]`)       | По индексу (`tuple[0]`)       | По ключу (`dict["key"]`)      | Нельзя по индексу / ключу     |
| **Тип элементов**  | Любые                        | Любые                         | Ключ: неизм. тип / Знач: любые| Неизменяемые типы             |

*\*Примечание про упорядоченность словарей: Начиная с Python 3.7 словари официально гарантированно сохраняют порядок вставки элементов.*
